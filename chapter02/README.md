## 本章核心思想

**整数运算和浮点运算具有不同的数学属性**——它们处理数字表示有限性的方式不同：整数运算是**精确**的（位级同构于模运算），而浮点运算是**近似**的（实数的有限精度表示）。

## 一、信息存储

### 1.1 虚拟地址空间与字长

- 大多数计算机使用**8位的块（字节 byte）**作为最小的可寻址内存单位
- 机器级程序将内存视为一个非常大的字节数组，称为**虚拟内存**
- 内存中每个字节都由一个唯一的数字标识，称为它的**地址**，所有地址的集合称为**虚拟地址空间**
- **字长(word size)**：指针的标称长度，也是虚拟地址空间的大小。32位系统的虚拟地址空间为 $2^{32} \approx 4$ GB，64位系统理论上为 $2^{64}$ 字节

### 1.2 C语言数据类型的大小

十进制和十六进制之间的转换（Python）：[d2h.py](d2h.py), [h2d.py](h2d.py)

C语言中各数据类型在64位系统上的典型大小：

| 类型            | 大小（字节） | 取值范围（补码）             |
| ------------- | ------------ | ------------------------------ |
| `char`        | 1            | $[-128, 127]$ 或 $[0, 255]$（符号性由实现定义） |
| `short`       | 2            | $[-32768, 32767]$              |
| `int`         | 4            | $[-2^{31}, 2^{31}-1]$          |
| `long`        | 4/8          | Windows(LLP64) 为4；Linux/macOS(LP64) 为8 |
| `int32_t`     | 4            | 固定32位（可移植）             |
| `int64_t`     | 8            | 固定64位（可移植）             |
| `char *`      | 8（64位字长）| 指针长度等于机器字长           |
| `float`       | 4            | IEEE 754 单精度                |
| `double`      | 8            | IEEE 754 双精度                |

[type_len.c](type_len.c)

> **可移植性原则**：应让程序对不同数据类型的确切大小不敏感。比如在32位系统把指针赋给 `int` 是OK的，但到了64位系统中会导致问题。推荐使用 `<stdint.h>` 中的 `int32_t`、`int64_t` 等固定大小的类型。
>
> **`char` 的符号性**：C 标准中 `char` 是有符号还是无符号是 **implementation-defined**（由实现定义）。x86/ARM 默认为有符号，但某些平台（如 ARM 编译选项）默认为无符号。需要明确时使用 `signed char` 或 `unsigned char`。
>
> **`long` 的大小差异**：这是 64 位系统上最常见的可移植性陷阱——Windows 使用 **LLP64** 模型（`long` 仍为 4 字节），而 Linux/macOS 使用 **LP64** 模型（`long` 为 8 字节）。跨平台代码应避免对 `long` 大小做假设。

### 1.3 字节序列与大小端

对于跨越多字节的对象，必须建立两个规则：
1. 对象的地址是什么？（答：所用字节序列中最小的地址）
2. 内存中如何排列这些字节？

**两种字节顺序（Endianness）**：

| 字节序 | 英文       | 规则                                 | 典型机器           |
| ------ | ---------- | ------------------------------------ | ------------------ |
| 小端法 | Little-endian | 最低有效字节排在最低地址（先存）     | x86、ARM（默认）   |
| 大端法 | Big-endian  | 最高有效字节排在最低地址（先存）     | 多数网络协议、SPARC |

使用强制类型转换来访问不同程序对象的字节表示：[show-bytes.c](show-bytes.c)

```shell
✗ ./show-bytes  12345
 39 30 00 00
 00 e4 40 46
 2c 5b c9 5b ff 7f 00 00
```

> 示例 `int 12345`（十六进制 `0x3039`）在小端机器上字节序列为 `39 30 00 00`（最低有效字节 `0x39` 排在前面）。第二行是 `float` 表示，第三行是指针值。

### 1.4 字符串表示

文本数据比二进制代码具有更强的平台独立性，因为字符串的表示本身做了抽象：字符串总是从给定地址开始，遇到 `0x00`（null终止符）结束，与字节顺序和字大小规则无关。

```c
show_bytes("12345", 6) → 31 32 33 34 35 00
```

**练习题2.7**：下面输出结果是 `0x61-0x66`，没有 `00`，因为 `strlen` 不计最后的 null：
```c
const char *s = "abcdef";
show_bytes((byte_pointer)s, strlen(s));
```

### 1.5 机器代码的平台相关性

查看机器代码可知，**二进制代码是不兼容的**（不同机器架构对应不同的指令集）：

```shell
hexdump sum.o
```

[sum.c](sum.c) 在 Mac 64 位上的字节序列：`00001d0 55 48 89 e5 89 7d fc 89 75 f8 8b 75 fc 03 75 f8`

### 1.6 布尔代数与位级运算

布尔代数的四种基本运算：

| 运算 | 符号 | 含义                       |
| ---- | ---- | -------------------------- |
| 与   | `&`  | 都是1才为1                 |
| 或   | `\|` | 至少一个1就为1             |
| 非   | `~`  | 翻转                       |
| 异或 | `^`  | 不同为1，相同为0           |

- **位向量（bit vector）**：一个很有用的应用是用来表示有限集合，对应的 bit=1 表示对应元素存在，有点 Bloom filter 的味道

### 1.7 掩码运算

位级运算的一个常见应用是实现**掩码运算**——掩码是一个位模式，用来选择性地保留或屏蔽某些位。

```c
~0          // 全1掩码（所有位都为1），比 0xFFFFFFFF 更具可移植性
0xFF        & x  // 取 x 的最低字节
~0xFF       & x  // 取 x 除最低字节外的所有位
```

> **可移植性建议**：使用 `~0` 得到全1的掩码，而非 `0xFFFFFFFF`，避免假设字长。

### 1.8 位运算应用练习

#### 习题2.10 - 不使用临时变量交换两个数

对任何一个向量 $a$，有性质 $a \oplus a = 0$。应用这一属性，可实现无需中间变量的交换程序：[inplace_swap.c](inplace_swap.c)

```c
*y = *x ^ *y;   // y 现在 = x_old ^ y_old
*x = *x ^ *y;   // x = x_old ^ (x_old ^ y_old) = y_old
*y = *x ^ *y;   // y = y_old ^ (x_old ^ y_old) = x_old
```

> 这种交换方式**不会有性能上的优势**，仅仅是智力游戏。利用异或的交换律即可证明。

#### 习题2.11 - 数组原地反转

承上题，利用上面的 `inplace_swap` 函数实现对调一个数组的元素：[reverse_array.c](reverse_array.c)

**源程序的陷阱**：当数组长度为奇数时，循环到中间元素时会出现 `first == last`，此时对同一个地址做 swap，由于 `*x ^ *x = 0`，会把中间元素置零！

**修复**：循环条件必须用 `first < last`（严格小于）而非 `first <= last`，避免对中间元素自交换。

#### 习题2.13 - 只用 bis 和 bic 实现位级或/异或

题意：`bis(x, m)` 对 `m` 中为1的 bit 把 `x` 对应位置1；`bic(x, m)` 对应地把 `m` 中为1的 bit 清0。**不是操作第 m bit**，而是操作 `m` 中所有为1的位。

- **或运算**：`bool_or(x, y) = bis(x, y)`（直接用 bis 即可）
- **异或运算**：利用恒等式 $x \oplus y = (x \& \sim y) | (y \& \sim x)$
  - 即"x 中不被 y 包含的位" ∪ "y 中不被 x 包含的位"
  - 实现：`bool_xor(x, y) = bis(bic(x, y), bic(y, x))`

[bis_bic.c](bis_bic.c)

### 1.9 C语言中的移位运算

| 运算       | 符号  | 规则                                       |
| ---------- | ----- | ------------------------------------------ |
| 左移       | `<<`  | 右端补0                                    |
| 逻辑右移   | `>>`  | 左端补0                                    |
| 算术右移   | `>>`  | 左端补最高位的值（符号扩展）               |

> **关键规则**：
> - 几乎所有的编译器/机器都会对**有符号数**使用**算术右移**；无符号数必然是逻辑右移
> - Java 中可以明确指定：`>>` 算术右移，`>>>` 逻辑右移 [Shift.java](Shift.java)
> - **注意运算优先级**：在C中加法比移位优先级高！`1 << 2 + 3` 实际等于 `1 << 5`，应该写成 `(1 << 2) + 3`。不确定就加括号。

## 二、整数表示

### 2.1 无符号数编码 B2U

$w$ 位向量 $\vec{x} = [x_{w-1}, x_{w-2}, ..., x_0]$ 到无符号数的映射：

$$B2U_w(\vec{x}) = \sum_{i=0}^{w-1} x_i \cdot 2^i$$

- 取值范围：$[0, 2^w - 1]$，记 $UMax_w = 2^w - 1$
- $B2U_w$ 是**双射函数**：每个位向量唯一对应一个无符号数，反之亦然

### 2.2 补码编码 B2T

最常见的有符号数表示方法（two's complement），最高位为**符号位**，权重为 $-2^{w-1}$：

$$B2T_w(\vec{x}) = -x_{w-1} \cdot 2^{w-1} + \sum_{i=0}^{w-2} x_i \cdot 2^i$$

- 取值范围：$[-2^{w-1}, 2^{w-1} - 1]$
  - $TMin_w = -2^{w-1}$（最高位为1，其余为0，如 4位时 `1000 = -8`）
  - $TMax_w = 2^{w-1} - 1$（最高位为0，其余为1，如 4位时 `0111 = 7`）
- **关键不对称性**：$|TMin_w| = TMax_w + 1$，这是许多微妙错误的根源
- $B2T_w$ 也是**双射函数**，相应的数和二进制向量唯一映射

> **可移植性原则**：C语言标准没有要求用补码表示有符号整数，但是几乎所有的机器都是这么做的。最大移植性的做法是利用 `<limits.h>` 中的 `INT_MIN`、`INT_MAX` 等典型取值范围，而不要自我假设。

### 2.3 B2U 与 B2T 的关系

从位模式角度观察：

$$B2U_w(\vec{x}) - B2T_w(\vec{x}) = 2^w \cdot x_{w-1}$$

即：**最高位为1时，无符号解释比补码解释大 $2^w$；最高位为0时，两者相等**。

### 2.4 有符号数与无符号数之间的转换

**核心原则**：强制类型转换时**位模式保持不变**，只是改变了解释这些位的方式——数值可能会改变，但是位模式不变。

$$T2U_w(x) = \begin{cases} x + 2^w, & x < 0 \\ x, & x \geq 0 \end{cases}$$

$$U2T_w(u) = \begin{cases} u, & u \leq TMax_w \\ u - 2^w, & u > TMax_w \end{cases}$$

理解有符号数和无符号数强制类型转化之间的数学关系：![](t2u.jpg)

[unsigned_cast.c](unsigned_cast.c)

**抽象理解**：之前负数 $x$ 的最高 bit=1 代表的是 $-2^{w-1}$，现在解释为无符号数时代表 $2^{w-1}$，所以增加了 $2^w$。

### 2.5 C语言中的隐式转换陷阱

- `printf` 没有任何类型信息，可以用 `%d` 输出 `unsigned` 类型，也可以用 `%u` 输出 `int` 类型 [printf_no_type.c](printf_no_type.c)
- C语言运算表达式中如果一个是有符号数一个是无符号数，会**隐式地将有符号转换为无符号**来执行运算。对标准算术运算可能差异不大，但对于**关系运算**就要注意了 [type_cast_op.c](type_cast_op.c)

```c
-1 < 0U      // 错！-1 被转为 UMax，结果为 false
```

### 2.6 数字扩展（Extension）

将一个 $w$ 位的数扩展为 $w+k$ 位，保持数值不变：

| 类型         | 规则                                       |
| ------------ | ------------------------------------------ |
| 无符号数     | **零扩展**：高位补0                         |
| 补码有符号数 | **符号扩展**：高位补符号位（最高位的值）    |

[zero_sign_extension.c](zero_sign_extension.c) — OSX 是小端，所以输出 `c7 cf ff ff`

**符号扩展正确性的证明**（利用数学归纳法，证明一位扩展正确即可）：

$$B2T_{w+1}([x_{w-1}, x_{w-1}, x_{w-2}, ..., x_0]) = -x_{w-1} \cdot 2^w + \sum_{i=0}^{w-1} x_i \cdot 2^i$$

$$= -x_{w-1} \cdot 2^w + x_{w-1} \cdot 2^{w-1} + \sum_{i=0}^{w-2} x_i \cdot 2^i$$

$$= -x_{w-1} \cdot 2^{w-1} + \sum_{i=0}^{w-2} x_i \cdot 2^i = B2T_w([x_{w-1}, ..., x_0])$$

关键点：$2^w - 2^{w-1} = 2^{w-1}$。证明过程见：![](sign_extension_prove.jpg)

### 2.7 数字截断（Truncation）

将 $w$ 位的数截断为 $k$ 位（丢弃高位）：

- 无符号数：$u' = u \mod 2^k$，截断后仍为无符号数，等价于模 $2^k$
- 补码数：$x' = U2T_k(T2U_w(x) \mod 2^k)$，先转为无符号，模 $2^k$，再转回补码解释

### 2.8 类型转换的顺序

C语言标准：把 `short` 转换成 `unsigned` 时，**先改变大小再完成从有符号到无符号的转换**。顺序为：`short → int → unsigned`。

## 三、整数运算

计算机的整数运算本质上是**模运算**——这是理解溢出、逆元、截断的关键。

### 3.1 无符号加法（模运算）

$$x +^u_w y = (x + y) \mod 2^w$$

溢出的最高一位被丢弃。**判断是否溢出**：结果反而比任一操作数小（无符号加法溢出当且仅当 $s < x$ 或 $s < y$）。

[uadd_ok.c](uadd_ok.c)

### 3.2 无符号加法逆元（阿贝尔群）

无符号数集 $\{0, ..., 2^w-1\}$ 在模 $2^w$ 加法下构成**阿贝尔群（Abelian group）**。每个元素 $x$ 的加法逆元：

$$\neg^u_w x = \begin{cases} x, & x = 0 \\ 2^w - x, & x > 0 \end{cases}$$

### 3.3 补码加法与溢出

补码加法在位级别上与无符号加法相同，但解释为有符号数：

$$x +^t_w y = U2T_w((x + y) \mod 2^w)$$

**三种溢出情况**：

| 情况         | 条件                                  | 结果           |
| ------------ | ------------------------------------- | -------------- |
| 正溢出       | $x > 0, y > 0, s \leq 0$              | 结果变负       |
| 负溢出       | $x < 0, y < 0, s \geq 0$              | 结果变正       |
| 不溢出       | 其他                                  | 正确           |

**判断溢出的技巧**：无论是否溢出，`x + y - x == y` 在位级别都成立。判断两个同号相加得到异号即为溢出。

[tadd_ok.c](tadd_ok.c)

### 3.4 补码的非（加法逆元 negation）

$$\neg^t_w x = \begin{cases} -x, & x > TMin_w \\ TMin_w, & x = TMin_w \end{cases}$$

**关键边界**：$TMin_w$ 的逆元是它自身（因为 $|TMin_w| = TMax_w + 1$，$-TMin_w$ 无法表示）。位级别理解：就是用 $2^w - x$ 按位运算。

字长4位示例：
- $3$ 的逆元：$10000 - 0011 = 1101 = -3$
- $-8$（TMin）的逆元：$10000 - 10000 = 10000 = -8$（不变）

在任何时候都要考虑 $TMax$、$TMin$ 这些临界情况！

理解加法逆元的哲学，对补码的逆就是补码的非 negation：![](additive_inverse.jpg)

### 3.5 减法的陷阱

**习题 2.32** [tsub_ok.c](tsub_ok.c)：判断 `x - y` 是否溢出。问题在于当 $y = \text{INT\_MIN}$ 时，$-y = \text{INT\_MIN}$（溢出！），`tadd_ok(x, -y)` 会导致错误判断。

正确做法是直接判断：正溢出（$x \geq 0, y < 0, x-y \geq 0$）、负溢出（$x < 0, y > 0, x-y \leq 0$）。

### 3.6 习题2.25 - 无符号运算的致命陷阱

问题在于无符号运算：`0U - 1 = UMax`，然后循环访问非法地址，导致 coredump。

```c
for (i = 0; i <= length - 1; i++) { ... }  // length=0 时，length-1=UMax
```

[sum_elements.c](sum_elements.c)

### 3.7 习题2.26 - size_t 的微妙

`size_t` 定义为 `unsigned int`。在做差和比较时会采用无符号运算，如果 `a < b`，则 `a - b` 是负数，会成为一个很大的无符号数，结果不正确。

[stronger.c](stronger.c)

### 3.8 乘法与截断

无论无符号还是补码，**乘法运算的位级表示都是一样的**：相乘，然后只截断为 $w$ 位，进而解释为无符号或补码形式。这样机器级的指令只需要一种，是**同构**的。

$$x *^u_w y = (x \cdot y) \mod 2^w$$

$$x *^t_w y = U2T_w((x \cdot y) \mod 2^w)$$

**判断乘法是否溢出** [tmult_ok.c](tmult_ok.c)：用更高精度的类型计算乘积，再比较。

### 3.9 乘以常数的优化

C编译器使用**移位、加法、减法**来消除很多乘以常数的情况，以减小计算代价：

| 乘法        | 优化为                              |
| ----------- | ----------------------------------- |
| `x * 2^k`   | `x << k`                            |
| `x * 14`    | `(x<<3) + (x<<2) + (x<<1)`          |
| `x * 7`     | `(x<<3) - x`                        |

> 这种优化使得"乘法比移位慢得多"的认知在现代编译器下被消解。

### 3.10 除以2的幂

除法不能像乘法那样直接用移位替代，因为：
- 整数除法默认**向零舍入**（round toward zero）
- 而算术右移是**向下舍入**（round down，向负无穷）

对于负数：`x / 2` 不等于 `x >> 1`。需要先加上"偏置"再移位：`(x + (1<<k) - 1) >> k`。

## 四、浮点数

### 4.1 IEEE 754 标准结构

$$V = (-1)^s \times M \times 2^E$$

| 字段         | 含义                           |
| ------------ | ------------------------------ |
| `s` (sign)   | 符号位，1位                     |
| `M` ( significand/mantissa ) | 尾数，二进制小数  |
| `exp` (exponent) | 阶码，对 $M$ 进行加权           |

**单精度（32位）布局**：`1位s | 8位exp | 23位frac`

**双精度（64位）布局**：`1位s | 11位exp | 52位frac`

### 4.2 三种编码情况

| 情况             | 条件                      | 阶码值 $E$              | 尾数 $M$                | 用途                |
| ---------------- | ------------------------- | ----------------------- | ----------------------- | ------------------- |
| **规格化**       | `exp` 全0或全1之外        | $E = e - Bias$          | $M = 1.f$（隐含的1）    | 最常见的数值        |
| **非规格化**     | `exp = 0`                 | $E = 1 - Bias$          | $M = 0.f$（无隐含1）    | 0、接近0的数        |
| **特殊值**       | `exp = 全1`               | —                       | `frac=0`: ±∞；`frac≠0`: NaN | 溢出、非数          |

- **偏置值 Bias**：单精度 $Bias = 127$，双精度 $Bias = 1023$
- **规格化的阶码范围**：单精度 $e \in [1, 254]$，对应 $E \in [-126, 127]$
- **非规格化数的巧妙设计**：使用 $E = 1 - Bias$ 而非 $-Bias$，使得从非规格化到规格化的过渡**数值连续**（最大的非规格化数与最小的规格化数之间间距一致）

### 4.3 浮点数的非均匀分布

- 浮点数在0附近**最密集**，在远离0的方向**越来越稀疏**
- 非规格化数填充了规格化数在0附近的间隙
- 这种设计使得在数值较小时仍能保持较高的相对精度

### 4.4 舍入模式

IEEE 标准定义了四种舍入模式：

| 模式                | 说明                                       |
| ------------------- | ------------------------------------------ |
| 向偶数舍入（默认）  | "最近舍入"，ties round to even             |
| 向零舍入            | 截断                                       |
| 向下舍入            | $-\infty$                                  |
| 向上舍入            | $+\infty$                                  |

**向偶数舍入（round-to-even）** 是默认模式：
- 通常舍入到最近的值
- 当正好处于中间值（如 `2.5`）时，舍入到**偶数**（如 `2`）
- 这种设计避免了统计偏差——一组数据的舍入期望无偏

### 4.5 浮点运算的属性

浮点运算在实数运算上的近似：

- 加法满足**交换律**：$x +^f y = y +^f x$
- 加法**不满足结合律**（单精度 float 下）：$(3.14 +^f 1\text{e}10) -^f 1\text{e}10 = 0 \neq 3.14 = 3.14 +^f (1\text{e}10 -^f 1\text{e}10)$
  - 左侧：$3.14$ 在与 $1\text{e}10$ 相加时因精度不足被舍入丢失，结果为 $1\text{e}10$，再减去 $1\text{e}10$ 得 $0$
  - 右侧：$1\text{e}10 - 1\text{e}10 = 0$，再加 $3.14$ 得 $3.14$
  - 注：此现象在单精度下成立；双精度下 $1\text{e}10$ 的 ULP 约 $1.9\text{e-}6$，远小于 $3.14$，不会丢失
- 乘法满足**交换律**，但**不满足结合律**和**分配律**
- 单调性：若 $a \geq b$，则除了 NaN 外 $a +^f x \geq b +^f x$（整数运算不具备此性质）

> **重要观察**：浮点运算不满足结合律，这是编译器优化时不能随意重组浮点表达式的原因。

### 4.6 C语言中的浮点数

- `float`：单精度，`double`：双精度
- C标准未明确要求使用 IEEE 754，但绝大多数现代机器都遵循
- 在 `int`、`float`、`double` 之间进行强制类型转换时，底层位模式会改变（这是与整数类型间转换不同的地方）
- `double → float` 可能损失精度（甚至溢出为 ±∞）
- `float/double → int`：值向零舍入，超出范围时行为未定义

## 五、核心思想总结

### 5.1 计算机运算的核心特征

| 性质           | 整数运算                       | 浮点运算                   |
| -------------- | ------------------------------ | -------------------------- |
| 精确性         | 精确（模运算）                 | 近似（受舍入影响）         |
| 结合律         | 满足（模运算意义下）           | 不满足                     |
| 分配律         | 满足（模运算意义下）           | 不满足                     |
| 溢出处理       | 静默回绕（wrap around）        | 产生 ±∞ 或非规格化数       |
| 与数学一致性   | 同构于 $\mathbb{Z}_{2^w}$ 环   | 近似实数                   |

### 5.2 抽象与上下文

- **同样的位模式，不同的解释**：这是贯穿本章（乃至全书）的核心思想
- 位级别运算（加、减、乘）对无符号数和补码数是**完全相同**的，差异仅在于解释
- 这正是**指令集架构(ISA)**作为抽象层的精妙之处

### 5.3 编程实践中的关键教训

1. **永远警惕无符号数**：与有符号数混合运算时会发生隐式转换，可能导致关系运算出错
2. **永远考虑边界**：`TMin`、`UMax`、`0`、`-1` 这些临界值最容易出 bug
3. **不要假设数据类型大小**：使用 `<stdint.h>` 中的固定大小类型
4. **不要假设字节顺序**：跨平台代码必须显式处理
5. **浮点比较要小心**：由于舍入误差，`a == b` 可能永远为 false，应使用 `fabs(a-b) < EPSILON`
